# Survey chains — reproducibility

The full 1-million-sample chain `.txt` files (Planck, DESI, KiDS, DES Y3,
joint cereal-bowl-weighted) are not stored in this repository because
they total ~1.1 GB and individual files exceed GitHub's 100 MB
per-file limit.

The summary statistics for each chain are preserved here:

* `_summary.json`     — original per-survey + joint chain summaries
* `_summary_v2.json`  — cereal-bowl-weighted + per-survey v2 summaries

Both files contain the per-survey best-fit χ², median χ², mean χ²,
n_samples, observable values, UM-derived predictions, and σ
specifications for full reproducibility.

To regenerate the chain `.txt` files from scratch, see the OSIRIS
companion repository:

```
github.com/joseph-shields/osiris
```

specifically the scripts under `tests_um/`:

* `survey_chains.py`         — original 4 surveys × 1M nuisance chain
* `joint_chain_weighted.py`  — cereal-bowl-weighted joint chain at 1M

Both scripts complete in seconds on CPU and reproduce the chains
bit-identically given the same random seed (default `20260430`).

The summary JSONs in this folder were extracted from the final
chains generated by those scripts and are sufficient for any
verification or cross-check task that does not require the
per-sample chain trajectories.
